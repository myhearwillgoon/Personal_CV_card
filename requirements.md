## 2. 非功能需求 (Non-Functional Requirements)

### 2.1 技术栈

| ID | 需求描述 | 规格 |
|----|----------|------|
| NF-001 | 前端技术 | HTML5 + CSS3 + JavaScript (ES6+) |
| NF-002 | 无框架依赖 | 使用原生技术，不引入React/Vue等框架 |
| NF-003 | CSS方案 | 原生CSS，可使用CSS变量 |
| NF-004 | 布局方式 | Flexbox + CSS Grid |
| NF-005 | 响应式 | Media Queries 断点设计 |

### 2.2 响应式断点

| 设备类型 | 断点范围 | 布局 |
|----------|----------|------|
| 桌面端 | ≥1024px | 三栏布局 (1:2:1 或 1:1.5:1) |
| 平板端 | 768px - 1023px | 两栏布局（头像简介上，视频下） |
| 手机端 | <768px | 单栏垂直布局 |

### 2.3 性能要求

| ID | 需求描述 | 目标值 |
|----|----------|--------|
| NF-006 | 首屏加载时间 | < 3秒 |
| NF-007 | 视频加载 | 懒加载，按需播放 |
| NF-008 | 页面大小 | 除视频外，HTML+CSS+JS < 500KB |
| NF-009 | 图片优化 | 头像压缩 < 100KB |

### 2.4 兼容性要求

| ID | 需求描述 | 目标 |
|----|----------|------|
| NF-010 | 浏览器支持 | Chrome, Firefox, Safari, Edge (最新2个版本) |
| NF-011 | 视频格式 | 兼容主流浏览器原生播放 |
| NF-012 | 移动端 | iOS Safari, Android Chrome |

### 2.5 视觉设计规范

| 项目 | 规格 |
|------|------|
| 主题风格 | 科技感、专业、简约 |
| 主色调 | 深蓝/靛蓝 (#2563EB) 或 深灰 (#1F2937) |
| 辅助色 | 浅灰背景、白色卡片 |
| 文字颜色 | 深色文字 (#111827)，次要文字 (#6B7280) |
| 字体 | 中文：系统默认（微软雅黑/PingFang SC），英文：Inter/Roboto |
| 圆角 | 卡片 8-12px，按钮 6-8px，头像 50% |
| 间距 | 8px 基准单位，卡片间距 16-24px |

## 6. 验收标准 (Acceptance Criteria)

### 6.1 功能验收

- [ ] 页面能在 Chrome/Firefox/Safari 正常显示
- [ ] 响应式布局在桌面/平板/手机上正确适配
- [ ] 头像、姓名、职位、邮箱信息正确显示
- [ ] 3个视频可在页面中播放
- [ ] 邮箱链接可点击并唤起邮件客户端
- [ ] GitHub Pages 链接可通过互联网访问

### 6.2 性能验收

- [ ] 首屏加载时间 < 3秒
- [ ] 视频播放流畅无卡顿
- [ ] 移动端滚动流畅

### 6.3 内容验收

- [ ] 个人信息准确无误
